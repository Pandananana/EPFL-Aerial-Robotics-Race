# epfl-drone-race

Perception for an EPFL drone-racing project. A Crazyflie with an AI-deck streams grayscale 324×244 frames over UDP; we want a model that detects rectangular LED gates in those frames. The scene is dark and the only reliable signal is the LED outline of each gate.

## Layout

- `recordings/<timestamp>/` — raw flight runs, treat as read-only. Each holds `img_NNNNNN.png` (grayscale) plus labelme JSON sidecars for labeled frames. Filenames collide across runs; never rename, the dataset layer handles uniqueness.
- `dataset/splits.json` — committed manifest, source of truth for which labeled frame is in which split. Regenerated by `tools/build_splits.py`.
- `tools/` — small utilities used during labeling (see Workflow below).
- `train.py` — exposes `predict_gates(image) -> list[(4,2) np.ndarray]`
- `test.py` — imports `predict_gates`, evaluates on the test split, reports precision / recall / F1 at a configurable IoU threshold. Polygon IoU is computed via mask rasterization with `cv2.fillPoly`.
- `crazyflie_fpv_example.py`, `test_connection.py` — hardware-side scripts for flying and verifying the radio link. Not part of the perception pipeline.

## Labeling conventions

- Each gate is a 4-point quad. Click order: **inner TL → TR → BR → BL** in the _gate's_ own frame, so corner identity survives rotation. This matters if we ever do PnP from the labels.
- Trace the **inner edge** of the LED frame (the hole the drone flies through), not the outer bloom or centerline.
- Skip a gate when near-edge-on, gate is partially blocked from view, or any corner is off-image.
- An image with no labelable gates is still saved with `{"shapes": []}`. This distinguishes "reviewed, empty" from "not yet reviewed" and `tools/finalize_no_gates.py` materializes that at the end of each labeling batch.

## Splits

Policy lives in `tools/build_splits.py:SPLIT_BY_RUN`:

- `20260513_112256` → test (an unrelated flight, no frame overlap with training data)
- other runs → train

Random splits across all labeled frames would give wildly inflated metrics because consecutive video frames are nearly identical — don't do this.
