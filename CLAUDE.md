# epfl-drone-race

Perception for an EPFL drone-racing project. A Crazyflie with an AI-deck streams grayscale 324×244 frames over UDP; we want a model that detects rectangular LED gates in those frames. The scene is dark and the only reliable signal is the LED outline of each gate.

## Layout

- `data/recordings/<timestamp>/` — raw flight runs, treat as read-only. Each holds `img_NNNNNN.png` (grayscale), `gate_positions.json`, and `measurements.csv`. Filenames collide across runs; never rename, the dataset layer handles uniqueness.
- `data/labels/seg/<timestamp>/img_NNNNNN.json` — labelme polygon sidecars, one per labeled frame. Mirrors the recordings layout so each image has at most one seg label at the matching path.
- `data/labels/pose/<timestamp>/img_NNNNNN.json` — 4-corner pose labels generated from the seg labels by `tools/convert_to_pose.py`.
- `data/splits.json` — committed manifest, source of truth for which labeled frame is in which split. Regenerated by `tools/build_splits.py`.
- `src/perception/models/<name>/` — each detector exposes `predict_gates(image) -> list[(4,2) np.ndarray]`. Current detectors: `hough_detector` (classical), `yolo_obb`, `yolo_seg`, `yolo_pose` (fine-tuned ultralytics checkpoints, weights in `best.pt` next to each `detector.py`).
- `src/perception/models/yolo_common/` — shared dataset builder that materializes the YOLO-format dataset on disk from `data/splits.json`.
- `tools/` — labeling utilities (see Workflow in README), plus `auto_label.py` (pre-label new recordings with YOLO seg), `visualize_preds.py` (overlay GT vs predictions), `seg_inference_video.py` (render a video of seg predictions over all recordings).
- `crazyflie_fpv_example.py`, `test_connection.py` — hardware-side scripts for flying and verifying the radio link. Not part of the perception pipeline.

## Labeling conventions

- Each gate is an LED rectangle with equal height, but variable width and pose.
- If the whole gate hole is visible in the frame, the polygon is 4 points. The number of points varies if the gate is partially out of frame (between 3 and 6+ points).
- The click order is clockwise starting from upper left.
- The **inner edge** of the LED frame has been traced (the hole the drone flies through), not the outer bloom or centerline.
- Skip a gate when near-edge-on, partially blocked from view, or any corner is off-image.
- An image with no labelable gates is still saved with `{"shapes": []}`. This distinguishes "reviewed, empty" from "not yet reviewed"; `tools/finalize_no_gates.py` materializes that at the end of each labeling batch.
